How It Works

A UUID (Universally Unique Identifier) version 4 is a 128-bit value that is almost certainly unique across space and time. It is generated using random bits, with a few fixed bits marking the version (4) and variant. This tool uses crypto.getRandomValues() for the random bits.

Example

A typical v4 UUID looks like 3f2504e0-4f89-41d3-9a0c-0305e82c3301 — 32 hex digits arranged in the pattern 8-4-4-4-12, where the first digit of the third group is always 4 and the first digit of the fourth group is 8, 9, a, or b.

Frequently Asked Questions

What is a UUID used for? ▾
UUIDs are commonly used as unique identifiers for database records, session tokens, distributed system keys, and more, since they can be generated independently without a central authority while remaining virtually collision-free.
Can two generated UUIDs collide? ▾
Theoretically yes, but the probability is astronomically small — with 122 random bits, you would need to generate billions of UUIDs per second for centuries before a collision became likely.
Is this a real RFC 4122 v4 UUID? ▾
Yes. The generator sets the version nibble to 4 and the variant bits per the RFC 4122 specification, using cryptographically secure randomness for all other bits.
